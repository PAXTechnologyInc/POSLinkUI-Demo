## 2. 副屏（A920Pro 等）

- 代码现状（演示工程）：`EntryActivity` 使用 `TransactionPresentation`、`SecondScreenInfoViewModel` 驱动副屏内容。  
- **要求**：副屏的边距、字号、是否双列等须走与主屏一致的 **Token + `DeviceLayoutSpec` 扩展**（如 `SecondaryDisplaySpec` 或同一结构内的 `role=SECONDARY`），**禁止**在 `Presentation` 内写死与主屏无关的一套 magic dp（除非该值为配置表字段）。

## 4. 与 Design Tokens 的关系

- `design-tokens.md` / `DesignTokens.kt` 存 **语义名**（如 `screenPaddingCompact`）及**基准档**数值。  
- **实际 dp/sp** 由 **`DeviceLayoutSpec` 档位**（或 `sw` 区间、最小高度）映射到语义名，业务只读 **规格 API**，不直接写 `16.dp` / `24.dp` 作为「全局真理」。  
- **还原度**：以设计稿指定 **基准机型** 作像素级对照；P0 其余机型验收 **SC-011**（不截断、可点击、无重叠），允许与基准存在由规格说明的阶梯差异。
