## 3. 验收门禁与性能阈值（T006）

| 指标 | 阈值 | 采集方式 |
|------|------|----------|
| SC-001 | 主流程 90% 交互 < 200ms 响应 | 手动或 UI 自动化 |
| SC-002 | CPU 下降 ≥ 25% | `top` 对比改造前后 |
| SC-003 | 内存峰值下降 ≥ 20% | `dumpsys meminfo` 对比 |
| SC-004 | 30 分钟温升降低 ≥ 2°C | 设备温感或外置测温 |
| SC-005 | 可控位点 100% 替换，不可控有降级 | 清单验收 |

**性能测量步骤**：执行 2.0 基线采集 → 完成动画改造 → 重复相同步骤 → 计算变化率，满足上述阈值即通过。

### 3.1 长时运行验证（T016, T017）

**30 分钟长跑步骤**：
1. 安装 `./gradlew installDebug`
2. 启动应用，进入主交易流程
3. 连续操作（页面切换、金额输入、确认）30 分钟
4. 每 5 分钟采集：`adb shell dumpsys meminfo com.paxus.pay.poslinkui.demo`、`adb shell top -n 1`
5. 记录 CPU 峰值、内存峰值、温升（若可测）

**门禁阈值**：CPU 较改造前下降 ≥25%，内存峰值下降 ≥20%，温升降低 ≥2°C。
