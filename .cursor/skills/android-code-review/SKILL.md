# Android 代码审查

## 审查顺序（风险高的先看）
1. **正确性**：行为是否符合需求与边界情况？
2. **支付安全**：幂等、敏感数据处理、回滚/超时行为。
3. **并发与生命周期**：回调竞态、后台线程更新 UI、重复进入风险。
4. **可维护性**：分层、命名、重复代码、隐蔽副作用。
5. **测试**：深度、是否防回归、关键场景是否缺失。

## 问题记录格式
- **严重程度**：致命 / 重要 / 一般
- **位置**：文件与符号
- **为何重要**：对用户的影响或资金风险
- **修改建议**：具体且改动面尽量小

## 必查项
- 交易状态有单一可信来源。
- 交易收尾路径上不得吞掉异常。
- 日志中不得出现敏感字段明文。
- 回调重试时不得重复发出最终交易结果。

## 通过条件
仅当致命与重要风险已解决，或已明确接受并附带缓解措施时，方可通过审查。
